# Halimaw Burger - Event-Driven Architecture Guide

This document explains the **Instant Alert System** implemented to replace inefficient "Polling" (timers). This approach ensures that different parts of the system (like the Kitchen and Admin Dashboard) update the very millisecond an order is placed.

## 1. The Core: `OrderNotificationService`
This is a shared service that acts as a "Broadcaster." It maintains a list of listeners (the panels that need updating) and notifies them when an event occurs.

```java
// Location: com.myapp.util.OrderNotificationService
public class OrderNotificationService {
    private static final List<Runnable> orderListeners = new ArrayList<>();

    // Panels (Cook, Admin) use this to sign up for alerts
    public static synchronized void subscribeToNewOrders(Runnable listener) {
        orderListeners.add(listener);
    }

    // The Cashier uses this to "Shout" to the rest of the system
    public static synchronized void notifyNewOrder() {
        for (Runnable listener : orderListeners) {
            // Platform.runLater ensures the UI update happens on the main thread
            javafx.application.Platform.runLater(listener);
        }
    }
}
```

## 2. The Broadcaster (Cashier)
Whenever a successful payment is processed, the Cashier signals the service.

**Implementation in `CashierController.java`:**
```java
private void processOrder(String paymentType) {

    if (orderSavedSuccessfully) {

```

## 3. The Listener (Cook Panel / Admin)
Panels that need real-time data "subscribe" during their initialization.

**Implementation in `CookController.java` or `DashboardController.java`:**
```java
@FXML
public void initialize() {
    // ... basic setup ...

    // Subscribe to instant notifications
    // Whenever a new order is notified, it will run the loadQueue() method
    OrderNotificationService.subscribeToNewOrders(this::loadQueue);
}
```

## Why this is better than Polling?
1. **Zero Latency:** There is no "5-second wait." The cook sees the order instantly.
2. **Efficiency:** The computer doesn't waste CPU/Database power checking for updates every few seconds when nothing is happening.
3. **Reliability:** Timers can crash silently. Events are triggered by real user actions, making the system feel more "alive" and responsive.
